# Plan: Implementar Inferencia de Perfil de Usuario (Issue #66)

## Objetivo
Crear un módulo `backend/app/user_profiler.py` que infiera el perfil de usuario (novato, intermedio, experto) basado en los primeros 2 turnos de la conversación, analizando vocabulario técnico, especificidad de preguntas y uso de siglas.

## Criterios de Aceptación (del Issue)
1. `infer_user_profile([{"role": "user", "content": "¿qué es un panel solar?"}])` → `"novato"`
2. `infer_user_profile([{"role": "user", "content": "¿cuál es el Voc del JinkoSolar 460W a 25°C?"}])` → `"experto"`
3. El perfil inferido se almacena en el `Dict` de sesiones junto al historial
4. La inferencia solo se ejecuta en los primeros 2 turnos; turnos posteriores reutilizan el perfil
5. Si el historial tiene menos de 1 turno, retorna `"intermedio"` como default

## Arquitectura Actual (Contexto)

### Componentes Relevantes
- **`backend/app/session.py`**: Gestiona sesiones en memoria con `SessionManager`
  - `sessions: Dict[str, List[ChatTurn]]` - Almacena el historial de turnos por session_id
  - `ChatTurn` model: `question`, `answer`, `timestamp`, `source_documents`
  - Métodos: `add_turn()`, `get_history()`, `get_context_string()`, `clear_session()`

- **`backend/main.py`**: Endpoint `/chat` (línea 407)
  - Recibe `ChatRequest(message, session_id opcional)`
  - Usa `session_manager.add_turn()` para guardar turnos
  - Inyecta contexto via `session_manager.get_context_string(session_id)`

- **`backend/tests/test_session.py`**: Tests para session manager

## Plan de Implementación

### Fase 1: Crear módulo `user_profiler.py`

**Archivo**: `backend/app/user_profiler.py`

**Contenido**:
1. Función principal: `infer_user_profile(history: list[dict]) -> str`
   - Input: Lista de diccionarios con `{"role": "user", "content": "..."}` (formato compatible con OpenAI)
   - Output: Uno de `"novato"`, `"intermedio"`, `"experto"`
   - Lógica de inferencia:
     * Analizar solo los primeros 2 turnos de usuario
     * Extraer features técnicos:
       - **Vocabulario técnico**: Lista de términos conocidos (Voc, Isc, MPPT, inversor, etc.)
       - **Siglas**: Detectar uso de siglas (kW, kWh, W, m², °C, etc.)
       - **Especificidad**: Presencia de valores numéricos, modelos específicos, marcas
       - **Preguntas genéricas vs. técnicas**: Palabras clave como "¿qué es", "¿cómo funciona" (novato) vs. parámetros técnicos

2. Función auxiliar: `_extract_technical_score(text: str) -> float`
   - Retorna score 0-1 basado en densidad de vocabulario técnico

3. Función auxiliar: `_extract_specificity_score(text: str) -> float`
   - Retorna score 0-1 basado en especificidad (modelos, valores, marcas)

4. Función auxiliar: `_extract_acronym_score(text: str) -> float`
   - Retorna score 0-1 basado en uso de siglas técnicas

5. Clasificación final:
   - Score combinado = (técnico × 0.4 + especificidad × 0.35 + siglas × 0.25)
   - `< 0.33` → "novato"
   - `0.33 - 0.66` → "intermedio"
   - `> 0.66` → "experto"

**Consideraciones**:
- Si `len(history) == 0`, retornar `"intermedio"` (default)
- Procesar solo preguntas del usuario (filter `role == "user"`)
- Usar max 2 turnos para evitar sesgos por turnos posteriores
- Implementar con lógica stateless (sin dependencias de sesión)

**Cambios en `main.py`**:

2. En la función `chat_endpoint()` (línea 407):

   - Verificar si el perfil ya existe: `existing_profile = session_manager.get_user_profile(session_id)`

   - El perfil puede usarse en futuras iteraciones para personalizar respuestas (fuera de scope de este issue)

3. El perfil se inferirá automáticamente cuando el usuario envía su primer mensaje (antes de agregar el turno a la sesión)

## Detalles Técnicos

### Vocabulario Técnico para Detectar (keywords)
```python
TECHNICAL_TERMS = {
    # Energía/Potencia
    'watts', 'vatio', 'kw', 'kwh', 'voc', 'isc', 'vmp', 'imp', 'pmpp',
    # Componentes
    'inversor', 'controlador', 'bateria', 'modulo', 'panel', 'ficha tecnica',
    # Fabricantes (sigla + modelo)
    'jinko', 'canadian', 'longi', 'bifacial', 'monocristalino', 'policristalino',
    # Parámetros
    'eficiencia', 'temperatura', 'degradacion', 'garantia', 'años',
    # Siglas comunes
    'mppt', 'pwm', 'dc', 'ac', 'voltage', 'amperaje',
}
```

### Métricas de Scoring
```python
# Technical Score
- Count technical terms / total words → normalized to [0, 1]

# Specificity Score
- Presencia de números (kW, 460W, 25°C) → +0.3
- Presencia de marcas/modelos específicos (JinkoSolar 460W) → +0.4
- Presencia de siglas técnicas (kW, m², °C, W) → +0.3

# Acronym Score
- Count siglas técnicas / total words → normalized to [0, 1]
```

## Decisiones de Diseño

| Aspecto | Decisión | Razón |
|--------|----------|-------|
| **Storage del Perfil** | En `SessionManager.profiles` (Dict en memoria) | Mantiene consistencia con sesiones actuales; facilita futura migración a BD |
| **Timing de Inferencia** | Al recibir primer mensaje (antes de agregar turno) | Aprovecha el historial existente; evita contaminar con respuesta del asistente |
| **Algoritmo de Scoring** | Combinación ponderada de 3 features | Simple, interpretable, fácil de ajustar pesos en futuro |
| **Formato Input** | Diccionarios `{"role": "user", "content": "..."}` | Compatible con OpenAI API; fácil de extender a LLMs futuros |
| **Default** | "intermedio" | Neutral; ni muy exigente ni muy laxo |
| **Máximo de Turnos** | 2 turnos | Evita sesgo por interacciones posteriores; balance precisión vs. rapidez |

## Riesgos y Mitigaciones

| Riesgo | Probabilidad | Impacto | Mitigación |
|--------|-------------|--------|-----------|
| Algoritmo de scoring impreciso | Media | Bajo | Validar con ejemplos reales en PR; ajustar pesos |
| Thread-safety en `profiles` Dict | Baja | Alto | Usar lock existente en SessionManager |
| False positives (novato detectado como experto) | Media | Bajo | Tests exhaustivos; ajustar thresholds |
| Performance: scoring en cada request | Baja | Bajo | Procesar solo primeros 2 turnos; operaciones O(n) simples |
| Integración con `/chat` incompleta | Baja | Medio | Testing extensivo; validar flujo completo |
